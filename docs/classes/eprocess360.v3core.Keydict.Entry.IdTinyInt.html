\eprocess360\v3core\Keydict\Entry

\eprocess360\v3core\Keydict\EntryIdTinyInt

Class IdInteger For use for specifying a remote row id.

The basic functions that make Keydict Entries work. Keydict Entries are containers for data that have their own validation, sleep, wakeup and value methods. Keydict Entries never specify SQL statements to store data in the database, however they are responsible for making sure the data stored is available as a string.

Entries can essentially be arrays. When using an Entry to store an array, the sleep and wakeup methods must be defined to handle the array serialization and unserialization. Extend the ArrayEntry class when going this route.
